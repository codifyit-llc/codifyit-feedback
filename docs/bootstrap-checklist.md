# CodifyIT Feedback Repo Bootstrap Checklist

Use this checklist when standing up `codifyit-feedback`.

## Repo settings

- Repo name: `codifyit-feedback`
- Visibility: `Public`
- Discussions: `Enabled`
- Wiki: optional
- Projects: optional
- Public Issues for feature tracking: avoid; if Issues remain enabled for maintainers, pin guidance that redirects public requests to Discussions

## Discussion categories

Create these categories:

- `Ideas`
- `Product Improvements`
- `Cross-Product Feedback`
- `Announcements`
- `How to Use This Repo`

## Pinned discussions

Pin these four discussions:

- `Welcome to CodifyIT Feedback`
- `What Belongs Here`
- `What Not to Post Here`
- `How Decisions and Routing Work`

## Templates and guidance

- Use the reusable intake template from `docs/discussion-template.md`
- Keep the status-prefix guidance from `docs/maintainer-response-prefixes.md` visible to maintainers
- Keep public language simple and avoid internal 6D or approval-gate terminology

## Launch check

- Repo URL added to CodifyIT public demand surfaces
- README published
- Categories created
- Pinned discussions published
- At least one maintainer has confirmed the response pattern
