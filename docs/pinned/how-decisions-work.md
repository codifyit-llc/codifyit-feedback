# How Decisions and Routing Work

This repo is for feedback and communication, not public project management.

When you post here, the usual path is:

1. A maintainer reviews the discussion.
2. We may ask for clarification.
3. We decide whether to decline it, keep it on a watchlist, or route it into
   internal review.
4. We reply publicly with the current visible status.

Public statuses:

- `[Needs Clarification]`
- `[Under Review]`
- `[Watchlist]`
- `[Routed]`
- `[Declined]`

If something is routed, that means it has moved into CodifyIT's internal review
or the correct product repo. That downstream work is not managed publicly here.
