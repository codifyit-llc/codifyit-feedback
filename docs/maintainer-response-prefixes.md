# Maintainer Response Prefixes

Use these prefixes in public replies:

## `[Needs Clarification]`

Use when the request is in scope but missing enough context that no decision
can be made yet.

Example:

`[Needs Clarification] Thanks for sharing this. I want to make sure we are
understanding the right problem before we review it further. Can you clarify
which product or workflow this is tied to and who is most affected today?`

## `[Under Review]`

Use when the discussion is in scope and has entered normal internal review.

Example:

`[Under Review] Thanks for the clear write-up. We have captured this for review
and are evaluating where it fits relative to the affected product area and the
current portfolio.`

## `[Watchlist]`

Use when the signal is useful but not ready to route forward yet.

Example:

`[Watchlist] We are keeping this visible because it points to a pattern worth
watching, but we are not routing it forward yet. Additional examples or impact
detail would help strengthen the case.`

## `[Routed]`

Use when the request has been accepted into internal review or a downstream
product path.

Example:

`[Routed] We have accepted this into the next level of review and routed it to
the appropriate internal destination. We will keep this discussion updated at
the decision level, but downstream implementation work will not be managed
publicly here.`

## `[Declined]`

Use when the request is out of scope, not aligned, or not a fit right now.

Example:

`[Declined] Thanks for taking the time to write this up. We are not routing
this forward from this repo because it falls outside the scope of public
feedback intake. If the request includes client-specific detail, please move it
to the private channel already established with CodifyIT.`
